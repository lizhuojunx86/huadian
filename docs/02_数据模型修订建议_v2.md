# 02 数据模型修订建议 v2

> 本文件是对架构文档 v1 §3 "L2 知识层" 的修订建议。
> 基于 `docs/01_风险与决策清单_v2.md` 中识别的风险点，给出 schema 级的具体改动。
> 保留的 v1 表不再重复，只列出**变更**与**新增**。
>
> 本文档不是最终 schema，是设计建议。实施由后端工程师 + DBA（DevOps）基于此文档生成 Drizzle schema 与 SQL 迁移脚本，并经架构师 + 历史专家评审。

---

## 一、变更总览

| 改动类型 | 数量 | 主要影响 |
|---------|------|---------|
| 新增表 | 17 | 主要在身份建模、引用、Account 层、审计 |
| 修改表 | 8 | persons / events / places / source_evidences / books / relationships / raw_texts / allusions |
| 字段类型变更 | 多处 | TEXT → JSONB（多语言）、POINT → GEOMETRY、单值 → 区间 |
| 删除表 | 0 | 无破坏性删除 |

## 二、核心改动

### 2.1 身份层拆分

#### persons（修改）
```sql
ALTER TABLE persons
  DROP COLUMN name_courtesy,
  DROP COLUMN name_art,
  DROP COLUMN name_posthumous,
  DROP COLUMN name_temple,
  DROP COLUMN aliases,
  ADD COLUMN slug TEXT UNIQUE NOT NULL,
  ADD COLUMN reality_status TEXT NOT NULL DEFAULT 'historical',
    -- historical / legendary / mythical / fictional / composite / uncertain
  ADD COLUMN birth_date JSONB,          -- HistoricalDate 结构
  ADD COLUMN death_date JSONB,
  ALTER COLUMN biography TYPE JSONB,    -- 多语言
  ADD COLUMN provenance_tier TEXT NOT NULL DEFAULT 'primary_text',
  ADD COLUMN deleted_at TIMESTAMPTZ;

-- 放弃单字段 name_courtesy/name_art 等扁平字段
-- 全部名号改由 person_names 表承载
```

#### person_names（新增，核心）
```sql
CREATE TABLE person_names (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  person_id       UUID NOT NULL REFERENCES persons(id) ON DELETE CASCADE,
  name            TEXT NOT NULL,
  name_pinyin     TEXT,
  name_type       TEXT NOT NULL,
    -- primary / courtesy / art / studio / posthumous / temple
    -- nickname / self_ref / alias
  start_year      INTEGER,
  end_year        INTEGER,
  is_primary      BOOLEAN DEFAULT FALSE,
  source_evidence_id UUID REFERENCES source_evidences(id),
  created_at      TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_person_names_search ON person_names USING GIN(name gin_trgm_ops);
CREATE INDEX idx_person_names_person ON person_names(person_id);
```

#### identity_hypotheses（新增）
解决"老子可能是李耳/老莱子/太史儋"问题。
```sql
CREATE TABLE identity_hypotheses (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  canonical_person_id   UUID NOT NULL REFERENCES persons(id),
  hypothesis_person_id  UUID REFERENCES persons(id),  -- 可能为 NULL（待确立）
  relation_type   TEXT NOT NULL,
    -- same_person / possibly_same / conflated / distinct_despite_similar_name
  scholarly_support TEXT,                 -- 学界支持度描述
  evidence_ids    UUID[] DEFAULT '{}',
  accepted_by_default BOOLEAN DEFAULT FALSE,
  notes           TEXT,
  created_at      TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_hypotheses_canonical ON identity_hypotheses(canonical_person_id);
```

#### disambiguation_seeds（新增）
```sql
CREATE TABLE disambiguation_seeds (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name            TEXT NOT NULL,
  person_id       UUID REFERENCES persons(id),
  dynasty_hint    TEXT,
  context_hint    TEXT,       -- 书名 / 卷名模式
  priority        INTEGER DEFAULT 100
);
CREATE INDEX idx_disamb_name ON disambiguation_seeds(name);
```

#### role_appellations（新增）
"高祖 / 太祖 / 先帝 / 陛下"等称谓词典。
```sql
CREATE TABLE role_appellations (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  appellation     TEXT NOT NULL UNIQUE,
  resolve_rule    JSONB NOT NULL
    -- e.g. {"type":"emperor_by_book","book":"史记","target":"person_id_xxx"}
);
```

### 2.2 事件双层（Event + Account）

#### events（修改）
```sql
ALTER TABLE events
  ADD COLUMN slug TEXT UNIQUE NOT NULL,
  ADD COLUMN canonical_account_id UUID,  -- FK 后补
  ADD COLUMN reality_status TEXT DEFAULT 'historical',

  ALTER COLUMN summary TYPE JSONB,
  ALTER COLUMN significance TYPE JSONB,
  ADD COLUMN provenance_tier TEXT DEFAULT 'primary_text',
  ADD COLUMN deleted_at TIMESTAMPTZ;
```

#### event_accounts（新增，核心）
```sql
CREATE TABLE event_accounts (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  event_id        UUID NOT NULL REFERENCES events(id),
  source_book_id  UUID REFERENCES books(id),
  narrative       JSONB NOT NULL,          -- 多语言叙述文本
  participants    JSONB,                    -- 参与者及其角色
  places          JSONB,                    -- 地点及其作用
  sequence        JSONB,                    -- 事件内子步骤时序
  source_evidence_ids UUID[] DEFAULT '{}',
  reliability_score REAL DEFAULT 0.5,
  is_canonical    BOOLEAN DEFAULT FALSE,
  created_at      TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_accounts_event ON event_accounts(event_id);
CREATE INDEX idx_accounts_canonical ON event_accounts(event_id) WHERE is_canonical;
```

#### account_conflicts（新增，替代 version_conflicts 的 Account 层增强版）
```sql
CREATE TABLE account_conflicts (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  account_a_id    UUID NOT NULL REFERENCES event_accounts(id),
  account_b_id    UUID NOT NULL REFERENCES event_accounts(id),
  conflict_type   TEXT,
    -- factual / chronological / interpretive / omission / attribution
  diff_summary    JSONB,                   -- 结构化差异描述
  analysis        JSONB,                   -- AI + 人工分析
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

### 2.3 引用层（核心新增）

#### mentions（新增，核心）
解决"不可沽名学霸王"隐指项羽这类问题。
```sql
CREATE TABLE mentions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  raw_text_id     UUID NOT NULL REFERENCES raw_texts(id),
  position_start  INTEGER NOT NULL,
  position_end    INTEGER NOT NULL,
  surface_text    TEXT NOT NULL,           -- 表面词形 e.g. "霸王"
  entity_type     TEXT,                     -- person / place / event / allusion / artifact / institution
  entity_id       UUID,                     -- 可空（未 resolve）
  mention_type    TEXT NOT NULL,
    -- explicit / implicit / allusion / metaphor / pun / pseudonym / title_only / anonymous
  confidence      REAL NOT NULL DEFAULT 0.5,
  inference_basis TEXT,                     -- rule / context / llm / human
  detector_version TEXT,
  human_verified  BOOLEAN DEFAULT FALSE,
  created_at      TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_mentions_rawtext ON mentions(raw_text_id, position_start);
CREATE INDEX idx_mentions_entity ON mentions(entity_type, entity_id);
CREATE INDEX idx_mentions_unresolved ON mentions(mention_type) WHERE entity_id IS NULL;
```

#### allusion_usages（新增）
```sql
CREATE TABLE allusion_usages (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  allusion_id     UUID NOT NULL REFERENCES allusions(id),
  mention_id      UUID NOT NULL REFERENCES mentions(id),
  semantic_shift  TEXT,
    -- faithful / extended / ironic / subverted
  notes           JSONB
);
```

#### intertextual_links（新增）
```sql
CREATE TABLE intertextual_links (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_raw_text_id UUID NOT NULL REFERENCES raw_texts(id),
  target_raw_text_id UUID NOT NULL REFERENCES raw_texts(id),
  link_type       TEXT NOT NULL,
    -- quote / paraphrase / rebuttal / imitation / parody
  overlap_text    TEXT,
  confidence      REAL DEFAULT 0.5,
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

### 2.4 文本层

#### raw_texts（修改）
```sql
ALTER TABLE raw_texts
  ADD COLUMN text_original TEXT,            -- 原始繁体
  -- raw_text 存简体（检索友好）
  ADD COLUMN variant_of_id UUID REFERENCES raw_texts(id),  -- 若为其他版本的变体
  ADD COLUMN deleted_at TIMESTAMPTZ;
```

#### books（新增）
v1 没有独立 `books` 表。新增后集中管理元数据。
```sql
CREATE TABLE books (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title           JSONB NOT NULL,           -- 多语言
  author_person_id UUID REFERENCES persons(id),
  dynasty         TEXT,
  genre           TEXT,
    -- official_history / unofficial_history / poetry / philosophy / classic / ...
  credibility_tier TEXT NOT NULL,
    -- primary_official / primary_unofficial / secondary_compilation
    -- / tertiary_reference / folk
  license         TEXT NOT NULL,
    -- CC0 / CC-BY / public_domain / proprietary / unknown
  authoritative_version TEXT,
  slug            TEXT UNIQUE NOT NULL,
  metadata        JSONB DEFAULT '{}'
);
```

#### text_variants（新增）
```sql
CREATE TABLE text_variants (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),

);
```

#### textual_notes（新增）
```sql
CREATE TABLE textual_notes (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  raw_text_id     UUID NOT NULL REFERENCES raw_texts(id),
  position_start  INTEGER,
```

#### variant_chars（新增，字典）

```sql
CREATE TABLE variant_chars (
```

### 2.6 时间层

时间不单独成表，但所有日期字段标准化为结构：
```typescript
// JSONB 结构
interface HistoricalDate {
  year_min?: number;          // BCE 用负数
  year_max?: number;
  month?: number;
  day?: number;
  precision: 'day' | 'month' | 'year' | 'year_range' | 'decade' | 'era' | 'unknown';
  reign_era?: string;
  reign_year?: number;
  polity_id?: string;         // 多政权并立时
  lunar_month?: number;
  lunar_day?: number;
  sexagenary_year?: string;
  season?: string;            // 孟春 / 仲夏
  original_text?: string;     // 原文中的时间描述
}
```

## 三、命名与 slug 规则

所有需要 slug 的实体：`persons / events / places / books / allusions / institutions / artifacts / polities`

Slug 生成规则（优先级降序）：
1. 有专有 slug（人工指定）直接用
2. 按 `{pinyin_of_primary_name}-{dynasty}-{disambiguator}` 生成
3. 冲突时附加 UUID 短码（8位）

历史专家负责维护稳定 slug 清单；一旦确立，修改需 ADR。
