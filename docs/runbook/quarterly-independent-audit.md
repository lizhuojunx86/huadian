# Runbook：季度独立审计（ADR-043 D1）

> 每季度（1/4/7/10 月中旬）执行一次。首个实例：[audit-2026-07-16](../audits/audit-2026-07-16.md)。
> 云端例行任务 `quarterly-independent-audit-reminder` 会在每季度 15 日提醒。

## 独立性三原则（比流程更重要）

1. **不采纳既往自评结论**：STATUS 真相表、决策日记、方向评估、d-route-progress-review 等只能作为**被审计对象**引用；一切结论以一手证据（git / 代码 / 文件 / 外部可达性）重建。
2. **编排移出架构师**：审计的分工设计、综合、定级不由承担架构师角色的会话执行（新开会话、以"独立审计员"身份进入，不先读 CLAUDE.md 角色分配）。
3. **对抗式核验**：FAIL 与 INSIGHT 级 finding 逐条派独立 agent 设法**驳倒**，被驳倒的丢弃、表述过头的修正后保留（首次审计驳倒率 1/20+，修正率 ~50%——核验层有真实产出）。

## 八维度模板（可按当期裁剪）

| 维度 | 核心问题 |
|---|---|
| 宪法合规 | C-1~C-25 逐条 vs 仓库现实；挂起条款恢复条件是否触发 |
| ADR 体系 | index 一致性；近期 ADR 承诺兑现；重大动作无 ADR 缺口；supersede 链 |
| 代码资产 | 测试宣称复验（精确数字）；领域隔离；依赖 pin；环境故障 |
| 文档-现实 | 真相表逐项一手复核；STATUS 时效；坏链；CHANGELOG 漏报/虚报 |
| 战略 | 投入 vs 宣称优先级（git 定量）；负空间是否防住当期漂移轴；**判据重定义复核（ADR-043 D2 必查）** |
| 安全隐私 | 脱敏完整性（人员字段全枚举法，勿用关键词清单）；secrets；身份反推面变化 |
| 对外输出 | 制品实态核验；漏斗诚实评估；发送吞吐 vs 内容产能 |
| 跨项目生态 | 注意力真实分布；仓外战略裁定与仓内叙事是否两本账 |

## 输出与落档

- 报告写入 `docs/audits/audit-YYYY-MM-DD.md`：概要（PASS/WARN/FAIL/INSIGHT 计数）→ P0 → 分维度 → 建议优先级（🔴/🟡/🟢）→ 审计自身局限
- 报告**不复写**任何敏感值（file:line 指位）
- 结论与架构师自评并列挂 STATUS（附录形式），不覆盖真相表——两套记录的分歧本身就是信号
- 整改分批走用户 ACK：🔴 立即 / 🟡 当月 / 🟢 择机

## 已知局限（诚实声明）

- 这仍是"过程外部化"而非"身份外部化"（同族 LLM 读同仓自述）；无法排除共同盲区
- 外部平台反爬（SSRN/知乎/蒲公英 403）使部分制品只能采信本地证据 + 时间线自洽
- 真正的身份外部化 = 请一位真人（如已有交流的同行）按本 runbook 抽查一次——机会出现时优先
