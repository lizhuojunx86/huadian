## 1. Context

### 1.1 触发

2026-05-17 用户在阅读《道德经》时提出：是否可以用华典智谱框架解构入库？

该问题不是"技术上能不能做"——技术上当然能；而是**D-route 战略下的案例选取问题**：开 case-3 对 *框架抽象* 有什么价值，还是只对"做完道德经数据"有价值？后者按 D-route §3.4 + §7 应当 sunset。

### 1.2 既有案例位置

| 案例 | 状态 | 框架抽象贡献 |
|------|------|------------|
| case-1 史记 | L3 主案例 / Sprint L→AA 累计 16 sprint dogfood | R1-R6 / V1-V11 / guard chain / sprint-templates / role-templates 全部源出 |
| case-2 中药提取 | Stage 3 进行中 / F-001~F-004 / schema v0.4 outline lock | 跨域验证（B-route 工业制造）/ F-001 跨域版本"分层制品"假设 / 应用层方法论 08+09 双文章 |
| case-3 候选 | **本 ADR 评估对象** | 待定 |

case-1 验证"框架在叙事性历史文本上 work"；case-2 验证"框架在工业制造批生产记录上 work"。case-3 的合理性 = "它能验证 case-1 + case-2 都没暴露的框架盲点"。

---

## 2. Decision

**登记《道德经》为 case-3 条件触发候选。现在不开案例。** 同时为框架抽象保留两条低成本路径：

1. **思想实验路径**（零成本 / 立即可执行）：在 `docs/methodology/03-identity-resolver-pattern.md` v0.3+ 中，把"道德经的概念实体如何被 R1-R6 处理"作为一段 cross-domain projection 写入，验证框架可移植性的边界，**不入库、不动 schema、不开 sprint**。
2. **正式 case-3 路径**（触发条件见 §4）：触发条件达成后，按 case-2 同样的 GO-A → GO-? 阶段制开新案例。

---

## 3. Rationale

### 3.1 道德经会逼框架往哪里长（= 抽象价值）

三个**史记案例 + 中药案例都不会暴露**的盲点：

**(a) 概念实体 vs 命名实体**

R1-R6 identity resolver 当前是 NER 思路（人/地/官/事件）。《道德经》的"实体"是"道""无为""自然""玄"——这些不是 NER，是**概念在不同章 / 不同注疏体系下的语义飘移**。框架要不要支持 *concept-as-entity*？这是 case-1 + case-2 都不暴露的盲点。

**(b) 版本作为 first-class citizen**

case-1 史记里"版本"是注脚级（中华书局点校本为主）。case-2 中药里"版本"是工艺规程迭代级（重要但有限）。

《道德经》中**版本就是数据本身**：同一句在帛书甲乙本和王弼本里可能完全相反（"大器晚成" vs "大器免成" vs "大器免[晚]成"）；郭店楚简甚至章序、是否分章都不同。

这会逼 audit / provenance 系统把"source version"从注脚提升到与实体同级。对 case-2 也有迁移价值（不同 GMP 版本下的工艺规程对比）。

**(c) 注疏作为二阶知识（knowledge-about-knowledge）**

王弼怎么注道德经 = 知识；陈鼓应怎么综合王弼 + 河上公 = 知识 about 知识。框架现在没有 layered commentary 抽象。

**关键映射**：这恰好是 case-2 F-001 三层制品假设（compliance / tentative_actual / ground truth）的另一个域投影——

| 维度 | case-2 中药 F-001 | case-3 道德经候选 |
|------|------------------|------------------|
| 表层 | GMP 合规叙事（批生产记录） | 通行本（王弼本 / 河上公本） |
| 中层 | 实际工序执行（部分未写 / 操作工经验） | 注疏传统（王弼注 / 河上公注 / 陈鼓应今注） |
| 深层 | 身体化经验（车间主任的排产 / 操作工的自我保护） | 出土文献（帛书甲乙 / 郭店楚简）+ 哲学原义重构 |

**如果跑通，F-001 就从 case-2 单点 finding 升级为跨域规律**，框架护城河变深。这是 ADR-036 评估的最大潜在 payoff。

### 3.2 道德经**不会**带来的新东西

- 时空 / 人物消歧 / 谥号字号分裂 / 关系网络 → case-1 史记已覆盖
- 大规模数据处理压力 → 5000 字全文，反而太小
- 工艺执行 / 身体化经验 / 反算博弈 → case-2 才有
- F-002 / F-004 类型的"含量上下限反算"现象 → 道德经不会触及

### 3.3 现在不开的核心理由

1. **F-001 在 case-2 内部都还未拍实**（v0.4 outline lock / 应用层文章 09 v0.2 全本但未 v1.0）。在 case-2 把 F-001 拍实之前去开 case-3，等于让 F-001 在两个"不完全成熟的案例"间相互验证，逻辑上是反 D-route §3.4（案例服务于框架抽象）的：抽象未成型，再开案例只是堆数据。
2. **case-2 主线 Path C 未完**（B 段 22/22 ✅ / A 段 16/100 / Stage 3 进行中）。新开案例会分散主线注意力。
3. **D-route 时间锚**：2027-04 才是"第 2 个跨领域案例"的官方锚点（CLAUDE.md §9）；目前那个槽位实际已被 case-2 占着，case-3 自然落在更后。
4. **替代路径成本极低**：§2.1 思想实验路径可以零成本捕获 80% 的框架抽象价值，不必正式开案例。

## 4. 触发条件（满足任一即重新评估）

case-3 道德经从"候选"晋升"启动"，需满足**以下任一条件**：

| Trigger | 描述 | 当前状态 |
|---------|------|---------|
| T-A | case-2 F-001 拍实 v1.0（应用层方法论 09 发布 v1.0 / D-route §6 2026-10 锚点） | ⏳ 09 当前 v0.2 全本 |
| T-B | case-2 Stage 5+ 全部完成 / Path C 关档 | ⏳ Stage 3 进行中 |
| T-C | Layer 1 框架代码抽象 v0.1 release（D-route §6 2027-01 锚点） | ⏳ v0.3 已 release / v0.5 candidates 累积中 |
| T-D | 第三方贡献者（CONTRIBUTING.md 渠道）主动提议《道德经》或同类哲学 / 注疏型文本作为案例并自带数据准备 | ❌ 暂无 |
| T-E | 思想实验路径（§2.1）在方法论 03 中跑通后，发现概念实体 / 版本 first-class / 注疏二阶 三个盲点中至少 2 个**需要真实数据才能验证**（即"用思想实验跑不通"）| ⏳ 思想实验路径未启动 |

**任一条件触发 → 架构师起草 case-3 启动 ADR（继承 ADR-036） → 用户 ACK → 起 case-3 GO-A**。

## 5. Alternatives Considered

| 选项 | 评估 | 否决理由 |
|------|------|---------|
| A. 立即开 case-3 | 用户的初始问题倾向 | §3.3 三个理由：F-001 未拍实 / case-2 主线未完 / 时间锚不到 |
| B. 永久 reject case-3 | 守严 D-route §7 Negative Space | §3.1 显示三个真盲点 → 永久 reject 损失太大 |
| C. 登记候选 + 思想实验路径 + 触发条件（**本 ADR**） | 折中 | ✅ 选择 |
| D. 开"道德经 mini case"（只做一章） | 看似低成本 | 案例 sprint 治理开销固定（GO-A inventory / brief / dogfood / closeout）/ 单章无法验证版本 + 注疏维度，不如思想实验路径 |

---

## 6. Consequences

### 正面

- ADR 索引 + STATUS.md 中保留候选位 → 未来翻到时不会"重新想一遍"
- 思想实验路径在方法论 03 起草时执行 → 零成本获得"框架可移植性"测试
- F-001 跨域映射表（§3.1 (c)）保留下来 → case-2 应用层文章 09 v1.0 起草时可作为"未来工作"段引用

### 负面

- 占用 ADR 编号 / ADR 索引一行 + STATUS.md 一行 → 文档维护成本 +ε
- 若 case-3 长期未触发，候选位会变成"僵尸条目" → 需要在每次 ADR 索引盘点时确认"trigger 状态"

### 风险

- **风险 1**：用户未来在 case-2 未完时仍提议立即开 case-3 → 缓解：本 ADR 已书面化"为什么现在不开" + 触发条件，可直接 reference
- **风险 2**：思想实验路径在方法论 03 v0.3+ 起草时被忽略 → 缓解：在 03 起草时把本 ADR 列入 "Related" 头，提示作者执行

---

## 7. Out of Scope

本 ADR **不涉及**：

- 道德经具体哪个底本（王弼 / 帛书 / 楚简）作为 case-3 起点 — 留给 case-3 启动 ADR
- 概念实体的 schema 设计 — 留给 case-3 启动后的 schema v0.1
- 注疏抽取 prompt / pipeline — 同上
- 任何对 case-1 / case-2 schema 的改动 — 本 ADR 不动既有 schema
