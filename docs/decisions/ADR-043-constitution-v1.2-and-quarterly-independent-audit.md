## 1. 修宪内容（v1.1 → v1.2）

| 条款 | 变更 | 理由 |
|---|---|---|
| C-17 | 任务载体从"仅 `docs/tasks/T-NNN-*.md`"泛化为三种可追溯载体（任务卡 / sprint brief 内嵌清单 / case 策略文档 GO-* 日志），并规定载体最低要素（任务边界、主导角色、用户 ACK 点） | D-route 后实际演化出的两种载体在功能上**优于**任务卡（有 ACK、阶段门、closeout），但宪法明文自 2026-04-29 起再未被遵守——审计判 FAIL。修宪使宪法与良性实践对齐；不回补历史任务卡 |
| C-9 | 加状态注：D-route 期间挂起；恢复条件 = 新服务/产品面上线前必须先接入 | 无新服务上线场景，字面要求空转 |
| C-11 | 加状态注：down 方案由 ADR-017（forward-only + pg_dump 回滚锚 + 4 闸门）等价实现 | 仓内无正式迁移文件，字面"down 方案"无从谈起，但回滚能力实质存在 |
| C-21 | 加状态注：D-route 期间挂起（前提 C 端产品线已入 Negative Space）；数据层保留；恢复条件 = C 端 surface 重新立项 | 原豁免依据"Phase 1 会做前端反馈链路"已被 ADR-028 推翻，宪法却维持同等优先级——豁免依据过期构成宪法债 |

**回滚路径**（修宪程序 §七 条件 2）：任一状态注的恢复条件触发时，删除对应状态注即恢复条款全效；C-17 如需收紧回单一载体，新 ADR + v1.3 即可。**无数据模型影响，无迁移脚本**（条件 4 不适用）。

## 2. 独立审计季度化机制（audit 建议 #16）

**背景**：审计核验确认"评估者=生产者"结构性盲区——真相表、日记、方向评估、止损线全部由同一架构师设计并填写，最尖锐实例是 ADR-039 中被评估方在判据失败时亲手重定义判据。audit-2026-07-16 是首次评估编排移出架构师之手（过程外部化），效果已被证明（P0 脱敏残留、双账本漂移等均为自评从未发现的问题）。

**决策**：
- **D1 季度独立审计**：每季度一次（1/4/7/10 月中旬），由不带项目上下文、不采纳既往自评结论的独立 agent 编排执行，方法与输出格式沿用 [runbook/quarterly-independent-audit.md](../runbook/quarterly-independent-audit.md)；报告入 `docs/audits/audit-YYYY-MM-DD.md`，结论与架构师自评**并列**（不覆盖）。
- **D2 判据重定义触发条款**：任何"对外可验证判据"的定义变更（ADR-039 类操作），默认在变更后的下一次独立审计中列为必查项——由非当事方复核"重定义是诚实校准还是移动球门"。
- **D3 例行提醒**：云端例行任务 `quarterly-independent-audit-reminder`（每季度 15 日）提醒启动，防止机制靠记忆维持。
