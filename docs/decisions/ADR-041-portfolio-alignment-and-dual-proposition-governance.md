# ADR-041: 项目组合档位对齐 + 双命题治理 + 路线图触发制降级

- **Status**: accepted（用户 2026-07-16 令实施 audit-2026-07-16 黄色批次；本 ADR 落档其建议 #5 / #6 / #8 / #11 相关项）
- **Date**: 2026-07-16
- **Owner**: 首席架构师（起草）/ 用户（决策）
- **触发**: [audit-2026-07-16](../audits/audit-2026-07-16.md) 两项核心发现——①"仓外已发生单主线战略重排（2026-07-06），huadian 仓内定位叙事未同步，双账本漂移"；②"case-2 构成半个未宣告的第二次 pivot，governance-as-ratification 而非 governance-as-constraint"

---

## 1. 背景：两本账

**仓外账本**（用户《个人创业机会评估 2026-07-06》，全盘扫描 15+ 项目后的正式裁定）：

> "SinoReg 单主线加注；traceguard 按 7/19 门冷静结账；量化划归'投资'科目；其余项目分级冻结/归档。"
> huadian 相关线（chinese-history-mcp / storyextractor 同组）："**维持声誉公共品；作为 AI 工程公开作品集间接给主线背书**。"

**仓内账本**（本仓 STATUS / D-route 路线图，截至本 ADR 前）：huadian 仍以"活跃主线"口径滚动，路线图保留 2026-10 / 2027-01 / 2027-04 全量锚点。审计判定：下一个进入本仓的 agent 会按旧口径分配全量路线图工作，与用户真实意图（medium 火力维持）冲突。

**注意力事实**（审计一手核验）：2026-07 huadian commit = 0；活跃极为 storyextractor（57）、Drug101China（29）、traceguard（26）。case-2 启动后全仓 52 commit 中框架代码仅 ~6%。

## 2. 决策

### A1 — 组合档位落档

huadian 在用户个人项目组合中的档位 = **声誉公共品 + 方法论兵工厂**：为 SinoReg 主线间接供给可信度（公开作品集、SSRN 学术制品、方法论文章），自身不承担商业化目标。质量底线不降：kb_forge 93 passed 不破、真相表照实更新、月度日记节律（C-24）维持。**"活跃主线"表述自本 ADR 起从 STATUS 移除。**

### A2 — 双命题治理（正式宣告，结束 governance-as-ratification）

承认本仓自 2026-05 起事实承载两个命题：

| 命题 | 内容 | 归属 |
|---|---|---|
| P1 | Agentic KE 工程框架（kb-forge + 方法论 00-07 + 史记冻结 dogfood 底物） | 本仓原命题 |
| P2 | 制药数据完整性审计方法论（case-2 / 方法论 08-10 / SSRN 7099138 / 蒲公英探针） | case-2 衍生，受众与 P1 几乎零重叠 |

P2 的**学术制品线**（预印、外审、论坛、CC BY 数据）留在本仓——它是商业线的"内容与权威臂"；P2 的**商业动作**（付费产品、客户接触）一律属 Drug101China/SinoReg 侧，本仓不承载。

**离巢判据**：蒲公英闸 2 开启（免费分通道报告实际交付）或出现首个付费线索 → P2 商业衍生部分迁出本仓独立运作，本仓保留学术制品与方法论并加指针。

### A3 — 路线图从日程制改为触发制

| 原锚点（D-route §6） | 处置 |
|---|---|
| 2026-10 双文章 v1.0 + 外审 | **保留**（P2 学术线主产出，当前领先） |
| 2027-01 kb_forge v0.1 release | **冻结为触发制**：出现第 1 个真实外部使用者才解锁工程化议程。"使用信号"定义：非 dependabot 的人工 issue/PR、以依赖或 fork 形式的真实使用、或第三方 domain example。依据：全部使用类信号为零（2 stars / issue·PR 100% dependabot / storyextractor 重写不引用），背书类信号（baojie、HelloGitHub）不算使用 |
| 2027-04 第 2 跨域案例 | 维持"已由 case-2/traceguard 半兑现 + 等真实触发"（ADR-038 口径不变） |

### A4 — ADR-028"核心交付物"表述修订

ADR-028 中"框架 = 项目的核心交付物"修订为：**"方法论 + 案例证据链为核心交付物，框架代码为触发式支撑。"** 与真实资源分配（~6% commit 投入框架代码）及 2026-06-10 方向评估的价值排序一致。ADR-028 本体不改（宪法：变更用新 ADR），其 header 加 forward-pointer 注。

### A5 — case-2 ↔ 商业线双侧桥接登记

本仓侧：case-2 公开制品（SSRN 7099138 / 蒲公英 thread-816324 / 方法论 08-10）登记为**医药商业线的信任资产**（对应《医药深挖 2026-07-06》"单人可信度三招"第 2 招——用公开研究替代客户案例），见 case-2 strategy §12。Drug101China 侧的对称登记由用户/该仓会话执行（本 ADR 不越仓修改），建议行文：`信任资产：SSRN 预印 7099138（批记录数据完整性检测方法）+ 蒲公英论坛帖 —— 引用于对外材料时用"公开研究"口径`。

### A6 — Negative Space 季度重审机制

审计发现 D-route §7 负空间清单"只防上一场战争"（枚举的全是 2026-04 的 C 端诱惑，对实际发生的漂移轴无覆盖）。自本季度起，每季度把"当前最想做但边际抽象价值存疑的 3 件事"写进候选负空间。**首批（2026-Q3）**：① case-2 第 4 篇应用文章（止损线 S1 已有，重申）② 为 SSRN 下载量做投放优化 ③ 翻译工序早于外审信号启动。

## 3. 后果

- ✅ 仓内外两本账合一：新 session 按"声誉公共品"档位分配工作，全量路线图不再是默认议程
- ✅ P2 不再是"寄生"：有名分、有边界（学术留仓/商业在外）、有离巢判据
- ⚠️ 若外部使用者出现（A3 触发），kb_forge 工程化按 ADR-031 议程解锁——本 ADR 是冻结不是放弃
- ⚠️ 本 ADR 引用了两份仓外文档（创业机会评估 / 医药深挖，均 2026-07-06）——它们不入本仓（含个人职业信息），关键结论已在 §1 §2 摘录自足
