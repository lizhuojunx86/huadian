## 1. 背景

case-2（中成药提取数据完整性审计）的数据底料是真实企业的批生产记录 OCR。2026-06-10 公开输出模式切换时发现：企业名、供应商、员工真名、批号、**药品批准文号（可经药监局反查企业）** 已随 5 月的 push 进入公开 git 历史（自 commit 84f93a1）。

## 2. 追认的既往决策（2026-06-11 → 06-13，用户当时已逐项拍板、仅未立 ADR）

| # | 决策 | 备选与理由 |
|---|------|-----------|
| D1 | 立即转 private 止血 | 无争议 |
| D2 | 37 文件约 2300 处批量假名化 + 6 项 git mv 重命名；**数值计算输入全保留原值**（三篇文章可复算性不受影响，仅切断数值↔产品/批号/企业关联链） | 备选"删除数据树"会同时摧毁 case-2 的可复算性主张 |

| D5 | **假名映射任何条件下不共享、不落盘**（仅存处置会话） | 映射表本身是最高危单点 |

## 4. 显式记录的相关决策（同期，一并留痕）

- **SSRN 实名投稿 + 作者身份统一**（2026-07-11，用户决策）：论文署名真实姓名、通讯统一 lizhuojun@gmail.com（弃 x86x86@gmail.com）、Research Integrity 按"披露曾任职于被研究企业（假名化）"口径填写。这是不可撤回的身份绑定，与假名化体系存在结构性张力。
- **残余风险（未在本 ADR 内消除）**：实名 + 利益冲突披露 + 数据指纹（批次时间窗/操作间编号/工艺时长）使"企业 A"对行业圈层低成本可推——审计判定这与"学术可信度三件套"结构上不可兼得。**该风险的接受/缓解决策留给专门 ADR**（audit 建议 #7：枚举三条反推路径 × 三类受众，闭环 case-2 strategy §7 Open Question 4），不在本 ADR 内草率合并。
