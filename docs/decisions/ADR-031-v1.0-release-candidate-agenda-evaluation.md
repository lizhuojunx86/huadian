## 1. Context

### 1.1 ADR-031 vs ADR-030 形态差别

| 维度 | ADR-030 (v0.3 release) | **ADR-031 (v1.0 release candidate)** |
|------|---------------------|-----------------------------------|
| 性质 | 触发条件已 5/6 ✅ → **决定立即 release** | 触发条件评估当前差距 → **决定不立即 release** |
| 主要输出 | v0.3 release timing + Sprint S+1 启动 | v1.0 候选触发条件锁定 + Sprint U+N 路径预测 |
| 决策模板 | "拆解条件背后价值 → 评估替代实现"（per ADR-030 §3 选项 B 论证）| "评估当前距离 + 锁定条件 + 预测时间线"（本 ADR）|
| 触发后续 | Sprint T = release sprint | Sprint U+N 持续 maintenance / 跨域 / methodology v0.2 cycle |

ADR-031 是**评估型 ADR**（vs ADR-030 触发型），首先承担"明确 v1.0 触发条件不漂移"的责任。

### 1.2 5 sprint zero-trigger 达成 = 评估时机成熟

per Sprint R retro §5.1 期待 + Sprint T retro §5.1 实证：

```
Sprint P (清债 / 1 会话 / 0 触发)
  ↓
Sprint Q (新模块 / 2 会话 / 0 触发)
  ↓
Sprint R (清债+meth / 1 会话 / 0 触发)
  ↓
Sprint S (eval+meth / 1.5 会话 / 0 触发)
  ↓
Sprint T (release+feature / 2 会话 / 0 触发) ← 5 个 zero-trigger 达成 ⭐
```

**5 种不同形态 sprint** 都 zero-trigger = framework v0.3 + 5 模块齐备 + maintenance + eval + release sprint **完整稳定信号**。

但 zero-trigger 是 framework "无 regression" 的证明，**不等于** v1.0 release 准备就绪。v1.0 比 v0.x release 标准更高 — 需要评估其他维度（API 稳定性 / 跨域案例方实证 / 第三方 review / methodology 成熟度）。

## 2. Decision

**采用决策：不立即 release v1.0。锁定 7 候选触发条件（§3）+ 评估当前距离（§4）+ 预测路径时间线（§5）。**

详细论证见 §3-§5。

---

## 3. v1.0 候选触发条件（7 项 / 锁定）

| # | 候选条件 | 说明 |
|---|---------|------|
| 1 | ≥ 5 模块稳定 + ≥ 2 release cycle (v0.x → v1.0) | 框架抽象首次完整 + release 节奏实证 |
| 2 | Maintenance + Release sprint 累计 ≥ 5 个 zero-trigger | framework 无 regression 信号 |
| 3 | API 稳定 ≥ 6 个月 (no breaking changes) | 用户可信 / 升级路径平稳 |
| 4 | ≥ 1 跨域 reference impl 完整 working code | 跨域 valid 实证（vs 仅 cross-domain-mapping.md design intent）|
| 5 | 第三方 review (≥ 2 person) 表示 production-ready | 独立质量保证 |
| 6 | 累计 v0.x patch 落地率 ≥ 95% | debt 几乎全清 / 无遗漏关键问题 |
| 7 | methodology 7 doc 全 ≥ v0.2 | 方法论文档群成熟 / 跨域 fork 案例方文档充分 |

## 5. 路径预测（Sprint U+N）

按各条件触发难度排序：

### 5.1 内部可控条件（Sprint U+N 自驱动 / 预估时间线）

| 条件 | 路径 | 预估时间 |
|------|------|---------|
| #6 累计 v0.x patch ≥ 95% | DGF-N-04 + DGF-N-05 中至少 1 项 land（2 项之一即跨过 95% 阈值；当前 24/26 = 92.3%，加 1 = 25/27 ≈ 93%；+1 跨域 reference impl 当作新 deliverable 加 v0.x 候选 / 复杂）| 不主动驱 / 等触发 |
| #7 methodology 7 doc 全 ≥ v0.2 | methodology v0.2 cycle = 7 doc 全部从 v0.1.x → v0.2（每 doc ~25-60 min cross-ref polish + 实证锚点 + 元 pattern 加段）| Sprint U+1 → Sprint U+N（约 ≥ 4 sprint cycles）|

### 5.2 半外部条件（部分 architect 可控）

| 条件 | 路径 | 预估时间 |
|------|------|---------|
| #3 API 稳定 ≥ 6 个月 | Sprint Q→T 累计 0 ABI 变化 / 时钟从 v0.3.0 release 起算 / **2026-10 月起达成** | **6 个月**（2026-04-30 + 6 = 2026-10-30）|

### 5.3 外部触发条件（不可主动驱）

| 条件 | 路径 | 预估时间 |

### 5.4 综合时间线预测

**最早 v1.0 release 触发**（乐观估算 / 所有条件并行达成）：

```

                                ↓
```

## 6. Consequences

### 6.1 立即生效

- Sprint U 完成后**不启动 v1.0 release sprint**（继续 Sprint U+N maintenance / methodology v0.2 cycle / 等触发）
- methodology/02 v0.1.1 §12 "5 模块齐备阈值" 不修改（仍是 v0.x 阶段终点判断标准）
- v1.0 release ADR (假设是 ADR-04X) 起草标准：本 ADR §3 7 触发条件中 ≥ 6 项达成（保留 1 项灵活度 / vs ADR-030 6/6 全达成模板）

### 6.2 触发条件锁定

本 ADR 锁定 v1.0 release 候选触发条件为 §3 的 7 项。后续 Sprint U+N 累计实证可能调整：
- 加新条件（如发现 framework v0.x → v1.0 还有未识别的"成熟度维度"）→ 起草 ADR-04X 加新条件
- 删 / 弱化条件（如发现某条件实证 over-engineering）→ 起草 ADR-04X 调整

### 6.3 跨域案例方接触优先级

per ADR-030 §4.3 + Sprint T residual debts §1.1，跨域 reference impl 不阻塞 v0.3 release 但是 v1.0 必要条件（per 本 ADR §3 #4）。

**建议主动 outreach**：
- 法律领域（per Sprint Q debt §1.1 推荐）— 业务 schema 简单 / 适合示范完整 5 pattern
- 学术 disambiguation 领域 — ORCID / DOI 等结构化 anchor / 跨外部 reviewer 友好
- 医疗领域押后（PHI 法务复杂 / 需要专项评估）

### 6.4 Sprint U+N 候选议程演化

| Sprint | 主题 | 触发条件 contribution |
|--------|------|--------------------|
| Sprint U (本 sprint) | ADR-031 + methodology/06 v0.1.1 + /07 v0.1 + /00 §2 sync | #7 部分（methodology 7 doc 完整 / 但仍 v0.1.x）|
| Sprint U+1 ~ U+3 | methodology v0.2 cycle (per doc / cross-ref + dogfood 实证) | #7 推进（每 sprint 让 1-2 doc → v0.2）|

## 7. Validation Criteria

本 ADR 的 "决策正确性" 验证 checklist（v1.0 release ADR 起草时回填 / 即 Sprint Z+? 评估）：

- [ ] v1.0 release ADR (ADR-04X) 起草时 §3 7 触发条件全部仍 valid（无 over-engineering 或 missing）
- [ ] §5 时间线预测准确性 ≥ 70%（最早 / 保守区间内 v1.0 release 实际发生）
- [ ] 路径预测中至少 1 个外部触发条件在预估时间内达成（非完全 stuck）
- [ ] 期间 framework v0.3 → v0.4 / v0.5 等 minor release 全部 ABI 兼容（per #3 6 个月 API 稳定）
- [ ] 期间累计 v0.x patch 落地率持续 ≥ 90%（不退步）

如 ≤ 3/5 验证项达成 → ADR-031 标记 partial-success / 在 v1.0 release ADR 时反思修正。
