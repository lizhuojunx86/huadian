## 0. 这个案例在做什么

把"前制药企业负责人脑中的隐性经验 + 几批真实批生产记录"用 huadian D-route 框架严谨地 codify 成一个**领域知识工程参考案例**，验证**"规程外的经验才是 quality-decisive"** 这一命题，同时为 huadian 框架的可移植性提供第二个跨领域证据。

**关键命题**（来自用户 Stage 1 输入）：
> "往往不写在工艺流程和操作规程中的内容才是实践中经验发挥的关键点。"

本案例 v0.1 范围：**只做提取工序**，三个产品（P1 / P3 / P2），每产品 6 批连续生产记录。

## 4. 阶段 0 已发现的有意义事实

1. **3 个产品提取工艺差异极大**：P1纯水煎、参乌 75% 乙醇浸泡 16h + 醇提 + 水提 + 醇沉、P2水提；同企业内对不同药材采用完全不同工艺路线，这本身就是用户提到的"不同溶剂下影响参数差异"的真实写照。
2. **P2规程明文给出双设备方案**（E5 vs E6），是规程内"设备形态影响"的直接证据，OCR 后看实际选用很关键。
3. **P1 B1 投料数量精确符合规程的 0.81 折算**（162 万 / 200 万 = 0.81）— 7 种饮片偏差 0%。说明"投料数量"这条线被严格执行，但工艺过程参数是否同样紧贴规程，尚待 OCR 验证。
4. **规程"暗参数"密度高**：每个产品规程都有大量"至适量"、"适量"、"未限定时长"、"未限定温度"，这些都是 invariants 的潜在战场。
