## 3. 检验报告样本结构（基于甘草饮片 L1 PDF）

每份检验记录 PDF 通常 **9-56 页**，结构分四段：

### 3.1 主页 1-3（高密度数据）

- **第 1 页 — 检验报告**：报告编号 / 品名 / 批号 / 规格 / 供货单位 / 检验依据 / 来货数量 / 检验日期 / 报告日期 + 检验项目 × 标准 × 结果 × 结论 全表 + 总结论 + 签名（报告人/复核人/批准人）+ 质量保证部红章
- **第 2 页 — 物料放行单**：质量保证部审核员 + 部长 + 受权人三级签批，是 QA 的最终决策记录
- **第 3 页 — 请验单**：仓储部发起的检验请求，含厂家批号 ↔ 进厂批号 映射

这 3 页是机打表格 + 红章 + 手写签名的混合形态，OCR 难度低，数据密度极高。**先抽 1-3 页就够建立每份 PDF 的核心数据。**

### 3.2 原始记录页 4-N（待评估）

剩余 6-50+ 页推测是各检验项目的**原始记录**：

- 性状原始记录（描述性）
- 显微鉴别记录（带显微镜照片或绘图）
- 薄层鉴别记录（带 TLC 板照片）
- 水分测定原始记录（烘箱+电子天平读数）
- 灰分测定原始记录（炽灼+冷却+称量）
- 重金属测定原始记录（含 ICP/AAS 仪器输出？）
- 含量测定原始记录（HPLC 谱图 + 积分数据）
- 农残测定（GC-MS 谱图）

这些原始记录页**才是 F-001 二阶检验的核心战场**——因为：
- 仪器谱图反向计算的难度比"填表"高一个量级（需要伪造谱峰形态/积分面积/保留时间）
- HPLC 含量值如果不一致，要看是仪器误差、人工读数、还是有意调整
- TLC/显微等"主观判读"项目如果都"完美符合"，就是嫌疑信号

**建议策略**：v0.1 先只抓主页 1-3，建立每份 PDF 的核心索引；待 6 批同物料数据汇总后再针对**含量测定原始记录**（HPLC 谱图）做深入 OCR。

### 3.3 工作量估算

- **主页 1-3 × 81 PDF = 243 页**，约等于半个 batch 的工作量，1-2 个 session 可完成
- **完整原始记录 × 81 PDF ≈ 1500-3000 页**，需要 5-10 个 session

## 4. F-001 二阶检验初步判读（基于甘草饮片 L1 这一份 / v0.2 双通道改写）

### 4.1 单批数据"舒适过线"检查 — 按上限/下限通道分组

#### 上限指标侧（standard 是 max；下偏方向 = downward / "清白即好"话语驱动）

| 指标 | 标准 max | 实测 | _pct_of_LOD | 解读 |
|------|---------|------|-------------|------|
| 水分 | ≤12.0% | 8.1% | 67.5% | 中间地带 |
| 总灰分 | ≤5.0% | 2.2% | 44% | 中间偏低 |
| 铅 | ≤5 mg/kg | 1 | 20% | 远 |
| 镉 | ≤1 mg/kg | 0.03 | 3% | 极远 |
| 砷 | ≤2 mg/kg | 0.1 | 5% | 远 |
| 汞 | ≤0.2 mg/kg | 0.02 | 10% | 远 |
| 铜 | ≤20 mg/kg | 4 | 20% | 远 |
| 药屑及杂质 | ≤3% | 1% | 33% | 中间 |

**8 个上限指标分布形态**：`[3, 5, 10, 20, 20, 33, 44, 67.5]`
**单批分散度**：高（极差 64.5 个百分点；横跨"极远"到"中间"）
**单批判读**：如果上限指标统一向下伪造，预期会都集中在 5-20% 的"清白舒适区"。当前 8 个指标分散度高，**倾向真实测量**。

#### 下限指标侧（standard 是 min；下偏方向 = upward / "高即好"话语驱动）

| 指标 | 标准 min | 实测 | _倍率_of_LOQ | 解读 |
|------|---------|------|-------------|------|
| 甘草苷 | ≥0.45% | 0.87% | 1.93 | 超 LOQ 0.93 倍 |
| 甘草酸 | ≥1.8% | 2.5% | 1.39 | 超 LOQ 0.39 倍 |

**2 个下限指标**：实测都明显高于 LOQ。
**单批判读**：F-002 揭示"含量高 = 质量好"话语会推动 upward 反向计算。**单批不能区分"真实优质材料"与"被向上算"**。这是 F-002 浮现后的主要修正：v0.1 把"超 LOQ 1 倍 = 真实证据"作为正面信号，v0.2 把它降为 ambiguous。

#### 单批 combined judgment

证据分裂：上限通道 → tentative_actual；下限通道 → ambiguous。整批 layer 暂归 `tentative_actual`，但需 6 批分布检验确认。

### 4.2 6 批数据分布检验设计 — v0.2 分通道双方案（替代 v0.1 单通道方案）

⚠️ **v0.2 关键改动**：v0.1 的"标准差 / 集中度 / 季节趋势"三件套适用于上限通道，但完全不适用于下限通道（因为下限指标在"高即好"话语下分布形态可能本就被 upward 推动，集中度高未必意味着反算，反之亦然）。

#### 4.2.1 上限指标 6 批检验（保留 v0.1 三件套 + 强化交叉验证）

针对每个上限指标（水分/灰分/重金属/杂质）独立检验：

1. **分布形态（核心）**：6 批的 `_pct_of_LOD` 在 [0, 100]% 区间内是否自然分散？
   - 自然分散（极差 ≥ 30 个百分点 / σ ≥ 10 个百分点）→ 倾向 actual
   - 高度集中在低位（如全部在 5-20% LOD）→ 倾向 compliance（统一向下伪造）
2. **指标间相关性**：跨 6 批，"水分_LOD%"vs"灰分_LOD%"vs"铅_LOD%" 是否相关？
   - 各指标互不相关（|r| < 0.3）→ 倾向 actual（真实物理量本就独立）
   - 各指标高度共变（|r| > 0.7）→ 倾向 compliance（同一只手在统一造数）
3. **lot 内一致性**：同 lot 的不同检验记录（如 L2 lot 在 6 批之间复用），实测值是否一致？
   - 一致 → 增信（同 lot 同样品，本就该一致）
   - 不一致 → 提示数据来源混乱

**判定函数**（伪代码）：
```
upper_channel_layer(indicator):
    if 极差 ≥ 30 and σ ≥ 10 and 与其他指标 |r| < 0.3:
        return "actual"
    elif 极差 < 10 and 集中在 5-20% LOD and 与其他指标 |r| > 0.7:
        return "compliance"
    else:
        return "tentative_actual"
```

#### 4.2.2 下限指标 6 批检验（v0.2 全新设计 — 关键变更）

针对每个下限指标（活性成分含量）独立检验。**不能简单套上限通道方案**，因为 upward 反算的产物可能本就分布"合理"。

**新增的三种检验**：

(a) **集中度检验（反向解读）**：6 批含量是否集中在单一"优质值"？
   - 如全部 0.85-0.90%（紧贴某个"行业暗共识值"）→ 增加反算嫌疑
   - 如自然分散在 0.50-1.20%（包含临界过线值如 0.46%）→ 倾向 actual
   - 注意：与上限通道相反，下限通道的"分散度高"反而更可能是真实

(b) **lot 时间/季节关联**：实际中药材含量受采收季、储存时间影响，应有季节性变化。
   - 6 批含量与 lot 进厂时间有合理关联（如夏采 lot 含量略高 / 储存 1 年后含量略降）→ 增信 actual
   - 与时间无任何关联且变异度低 → 反算嫌疑加深
   - 数据：进厂批号末尾两位通常含月份信息（L1 = 26 年 1 月入厂）

(c) **下游相关性（最强证据）**：来料含量 ↔ 批生产记录的提取得率 / 成品含量
   - 高来料含量 lot → 提取后浓缩液体积/比重应有提升 / 成品含量应同向变化 → 倾向 actual
   - 来料含量与下游指标完全无关 → 反算嫌疑（来料数字与真实工艺脱钩）
   - 注意：下游成品含量本身也可能反算（F-001 表层），所以这条要结合多个下游证据交叉看

**判定函数**（伪代码）：
```
lower_channel_layer(indicator):
    if 自然分散 and 与 lot 时间合理关联 and 与下游指标合理相关:
        return "actual"
    elif 高度集中在"暗共识值" and 与时间无关 and 与下游无关:
        return "compliance"
    else:
        return "tentative_actual"
```

#### 4.2.3 整批 combined judgment

```
batch_layer = combine(
    upper_channel_layers: dict[str, layer],
    lower_channel_layers: dict[str, layer]
)
```

简化规则（v0.2 起步版）：
- 两通道都 actual → batch_layer = actual
- 两通道都 compliance → batch_layer = compliance
- 一边 actual 一边 compliance → batch_layer = mixed（关键发现：哪些指标真做了哪些没做？）
- 任何 tentative → batch_layer = tentative_actual

### 4.3 v0.2 命题中的 layer 判定 — 分通道版

来料检验数据**暂归 layer = "tentative_actual"**。最终归层取决于 §4.2 的双通道分析结果：

| 上限通道判定 | 下限通道判定 | 整批 layer | 案例含义 |
|------------|------------|-----------|---------|
| actual | actual | **actual** | F-001 表层局限只在批生产记录侧；来料检验是中层 ground truth |
| actual | compliance | **mixed** | 上限指标真做（合规驱动够强）；下限指标向上反算（话语驱动）— **F-002 命题获完整证据** |
| compliance | actual | mixed-反向 | 极少见，需重新审视 |
| compliance | compliance | **compliance** | 来料检验整体也是反向计算 — F-001 战场扩大，需要更深的 ground truth 层（HPLC 谱图原始记录、Stage 5.5 多模态采集）|

**最大概率预期**（基于 F-002 命题）：上限 actual + 下限 mixed/compliance → 整批 mixed。这正是 F-002 价值最高的发现路径。

## 5. Schema 扩展提议（v0.1 → v0.2 触发）

新增顶层 section：

```jsonc
"incoming_qc_来料检验": IncomingQCRecord[]
```

每条 IncomingQCRecord 字段已在 batch-record.json 中实例化（甘草饮片 L1），关键字段：

- 报告基本信息：report_id / 物料代码 / 进厂批号 / 厂家批号 / 供货单位 / 规格 / 来货数量 / 检验日期 / 报告日期
- 检验项目：性状 / 鉴别[] / 检查{水分/灰分/重金属/二氧化硫/杂质/农残} / 含量测定[]
- 每个数值字段必须含：标准 + 实测 + `_pct_of_LOD`（上限指标 / 派生）或 `_倍率_of_LOQ`（下限指标 / 派生）+ 结论
- 派生字段 `_F-001_F-002_indicators_v0.3`（v0.2 之 batch-record 中已实例化）：
  - `上限指标侧_证据`：指标数 / `_pct_of_LOD_分布` / 分散度 / `_judgment_单批` / `F-002 偏向: downward` / `_layer_倾向`
  - `下限指标侧_证据`：指标数 / `_倍率_of_LOQ` / `_judgment_单批` / `F-002 偏向: upward` / `_layer_倾向`
  - `_combined_judgment_单批`：单批整体判读
  - `_judgment_v0.3_strategy_分通道`：6 批检验时的具体执行策略（上限/下限分别落到 §4.2.1 / §4.2.2）
- 派生字段 `_lot_闭环`：批号 ↔ 批生产记录的精确映射 + 本批使用率
- 派生字段 `_layer_classification_v2`：`tentative_actual` / `actual` / `compliance` / `mixed`
- 附属文档：物料放行单 / 请验单（form_id 不同但同样在 PDF 中）

完整 schema v0.2 文档将在累积 ~5 份不同物料 OCR 后正式起草，避免过早 lock 错误结构。⚠️ **schema-v0.1 主文档当前仍为 lock 状态**，本节是 incoming_qc 侧的 v0.2 候选 patch；正式升级仍需用户批准。
