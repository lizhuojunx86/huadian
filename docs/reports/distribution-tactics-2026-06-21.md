## 一、这两周的诚实读数（3 句话）

1. **只有"管道通了"信号，没有"质量"信号**：首个 star、23 独立访客、3 次知乎引流——统计上全是噪声；328 clones 几乎都是 CI/爬虫，当 0 看；shiji-kb 2 天无回复是陌生 Discussion 的默认态，不是拒绝，别读成信号。
2. **唯一真信号**：第 4 篇拿去回答问题后的 **5 个收藏**（高成本动作 = "我以后要用"）。但 n=5，只能说"不是零"，不能说"成立"。
3. **不要过度操作**：这是 2-5 年的慢游戏，你在用 2 天粒度检视——采样频率比信号变化快两个数量级，看到的 99% 是抖动。

> "回答 70 > 专栏 40"：方向对（接入已有需求 > 对空发布），但这是**渠道机制**决定的常识，不是这条 n=1 数据证明的。别过度拟合成"该多写回答"然后下周写 8 个回答用 n=2 验证 n=1。

## 二、「回答式分发」操作手册（本阶段主战术）

**机制依据（已核实）**：0 粉时推荐池+搜索是唯一入口；知乎专栏推荐权重仅为回答的 1/5~1/10，且不参与问题页答案排名；回答挂靠已有浏览量的问题，按"点赞率"（赞/浏览）而非绝对赞数排序，对低赞新号极友好。换句话说：把内容从专栏换成回答 = 换到一条宽 5-10 倍的赛道。

**本周优先打的方向（按"立刻见效"排序）**：
1. **③ 框架被"抄"了之后** → 真实长青问答页 [zhihu.com/question/58413890](https://www.zhihu.com/question/58413890)（"没有 Fork 而是直接拷贝代码是否算抄袭"）。**第一个打**——4 篇里唯一锁定的真实问题页，你"被复刻=验证可移植性"视角差异化强。
2. **④ LLM 实体为何不能自动合并** → 复用已实测成功路径。站内搜 `LLM 实体去重`、`同名实体 消歧`、`知识图谱 实体对齐 LLM`，挑浏览高、现有答案差的挂答。
3. **② Claude Code 多 Agent 组队** → 站内搜 `Claude Code 多 agent`、`一个人 AI 团队 开发`（当下最高搜索量入口）。
4. **① agent 死循环/硬中断** → 站内搜 `agent 死循环`、`智能体 无限循环 终止条件`。

**写法与风控**：拆成 3-5 个"问题答案"别整篇搬运；**一个回答只放一个外链**、放底部配一句上下文、优先反链自己专栏再导 GitHub；长效引流放个人简介；**绝不刷赞/拉群求赞**；新号按真人节奏养（每天 2-3 条分时段）。可并行同步 5-8 条到 **SegmentFault 思否问答**。

## 三、节律（把检视频率降下来）

- **下一篇**：按原计划写，**别动选题/人设/风格**（样本量=4，任何"优化"都是拟合噪声）。
- **检视频率**：数据 **每 2-4 周看一次**，策略 **每季度调一次**。停止每 2 天看 traffic/star。
- **两个日历事件**（到点自然兑现，别催）：HelloGitHub 月底出刊 / 阮一峰周刊（OPEN 未选）。
- 两周后唯一该问：**"我有没有按计划产出 + 接入需求"**，而非"数字涨了没"。衡量用"问题页浏览/赞率"，不用"专栏阅读"。

## 四、【不做清单】

- ❌ 不因"回答>专栏"就 all-in 问答、放弃专栏长内容（长内容是 SEO+可信度资产，半年起效）
- ❌ 不再催 shiji-kb / 阮一峰 / 任何同行；最多 2-3 周后用**新作品**自然再触达
- ❌ 不因 4 篇数据差就改选题/人设/风格
- ❌ 不新增第 5 个平台（B站/公众号/小红书）
- ❌ 不天天看 GitHub traffic/star
- ❌ 不为互动蹭热点或写标题党（污染"严肃工程"定位）
