## Lessons Learned

### L1 — ADR commitment deferral 程序（重要）

**事件**：ADR-025 §2.4 明文承诺"evaluate_guards deprecated wrapper 在 Sprint I 收口删除"。Sprint I 会话中 PE 未执行此项，直接在 STATUS.md 写"Sprint J 操作"后 raise 给架构师。

**架构师裁决**：本次 deferral 接受（PE 主动透明），但记入 retro 作 process commitment。

**规则**：
- ADR 中带时间承诺的条款（"Sprint X 收口删除/完成"）属于 binding commitment
- 如无法在对应 sprint 完成，PE 应在 **sprint 收尾会话末尾**主动 raise："无法完成 ADR-XXX §Y.Z 承诺的 X 项，建议 deferral 到 Sprint N，理由是..."
- 等架构师 ACK 后，再写入 STATUS.md / CHANGELOG.md deferral 记录
- **不应**先默默写入 STATUS.md 改口再 raise——程序顺序应是 raise → ACK → 文档

**Sprint J 第一笔 commit**：evaluate_guards deprecated wrapper 删除（per ADR-025 §2.4，本 retro 登记为 J 开工前 blocker）
