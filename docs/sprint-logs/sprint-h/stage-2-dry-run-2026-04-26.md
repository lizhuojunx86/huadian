**根因分析**：检查 `惠公` entity 在 DB 中的实际 dynasty 值——可能因
NER 阶段归类为"战国"而非"春秋"（entity 内含晋惠公/秦惠公 mention 混

ADR-025 §4.2 第三条已知局限：**entity-level 污染** dynasty filter 无能为力，
需 mention-level 消歧（V12 候选）。

无回滚必要——guard 行为与 ADR §4.2 一致。
