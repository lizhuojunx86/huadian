## 2. 关键元决策（按时间）

### 2.1 D-route §6.2 案例优先级判定（GO-A 启动时）

**上下文**：case-2 启动时是 ADR-028 §2.3 Q4 ACK 后的首次"案例服务于框架"原则检验。

**关键判断**：case-2 数据完整度（B1 100 页 / 6 批 / 22 物料）vs 框架抽象产出 = **优先后者**。即使数据未全量 OCR，只要框架抽象产出（fundamental findings / methodology 文章 / ADR）已经成熟，可以推进下一阶段。

**实证**：4 个 fundamental findings F-001~F-004 全部跨案例可迁移 / 双文章 08+09 配对实证完整 / 2 个 case-2 ADR（033/034）/ 即使未来 case-2 数据补全 84 页剩余批生产记录，框架抽象产出**已经独立成立**。

**对未来的启示**：D-route §6.2 不是抽象原则 / 是具体的可操作判断 — 每个 stage 启动时问"这一步对框架抽象有什么独立价值"。

### 2.2 F-001 命题精细化路径（v0.1 → v0.4，7 天 4 个版本）

**精细化轨迹**：
- v0.1 (5/8 Pilot)：第一感是"严格执行"→ 用户答 Q-001/002 后修正为"反算填表 + 应付检查"
- v0.2 (5/9 来料检验): + F-002 双通道反算上下文（含量下限 upward 偏向）
- v0.3 (5/10): + F-003 子通道分裂（main/extract/minor 量级影响反算难度）+ ADR-034 schema v0.3
- v0.4 (5/10 GO-E-α): + 表层三类参数（输入/输出/衍生）+ 中层运筹层 + 深层身体化经验

**元决策反思**：F-001 命题的精细化由**用户 5 题访谈触发** / 不是从数据本身推断。这是案例方法论的**核心实证**：

> 领域专家访谈是 F-001 命题成立的必要条件 — AI 知识工程仅从记录数据**无法判读** σ ≈ 0 是"工艺优秀"还是"反算"。

这条 insight 后来成为 09 文章 §6 + §7.3 的方法论论断。一句话：**ground truth 校准必须包含访谈环节 / 这是本方法论与传统统计审计的根本性差异**。

### 2.3 F-004 命题浮现的元决策（GO-E-α → GO-G-γ）

**初始疑虑**：F-004 候选浮现于 GO-E-α / Q-027 用户答案后，当时疑虑 "F-004 可能只是 F-002 的特例"（同是反算 / 都是上下限偏向）。

**关键证据**：GO-G-γ Phase 2 收集P2 6 批数据后 — F-004 跨产品 σ 同量级（P1 σ=0.0223 / P2 σ=0.0275）+ 紧贴位置不同（下限 40% vs 标杆 38.3%）— **同一种博弈论行为模式跨产品复现 + 紧贴位置随规程设计变化** = F-004 是**独立命题** + 命题深化为"最不引人注目的安全位置"。

**元决策反思**：候选命题成立的"双产品验证" = 比"单产品深挖"更高 ROI。F-004 X1 finalized 用了 1 个 phase（P2 6 批）/ 验证完整。

**P3 X2 决策**：跨企业访谈（X2）维持 pending — 用户 5/14 答"人脉不确定"/ 选承认局限性而非强行推进 / v0.3 §8 学界审稿前明示 documented unresolvable。

### 2.4 双文章配对节奏决策（GO-F-α → GO-J-ε）

**两步走 vs 一步走**：
- GO-F-α (5/12) 起草 08 v0.1 outline 时同步识别 F-001 v0.4 命题成熟 → 应有第二篇文章
- GO-H-δ (5/12) 决定双文章配对结构（08 机制 / 09 结构）+ 写 09 v0.1 outline
- GO-I-ε (5/14) 09 v0.2 §2-§4 章节展开
- GO-J-ε (5/14) 09 v0.2 §5-§10 章节展开 → v0.2 全本完成

**元决策反思**：先 outline + 后展开的两步走（v0.1 → v0.2）比"一步直接写完整稿"**节奏更稳**。原因：

1. outline 阶段先锚定论点 + 章节结构 + 实证锚点 → 章节展开时不会偏移
2. 两步之间可以让用户预审 outline 是否论点清晰（虽然本案未实施）
3. 进度跟踪表（§13）能可视化"还缺什么"

**对未来的启示**：v0.3 完整初稿应保持同样的两步走 — 先精炼 4000-6000 字目标的"章节骨架预判"再展开。

### 2.5 写作"领先 vs 节制"的张力（GO-J-ε 后反思）

**当前状态**：09 v0.2 全本完成（950 行 / 67.2 KB）/ D-route §6 2026-10 v1.0 路线图领先 ~6-8 周。

**张力**：是否应当继续加速推进 v0.3 完整初稿？还是节制 / 留时间给用户审稿 + 学界外审？

**当前决策**：选**节制** — 本月（5 月）不启动 v0.3 完整初稿 / 2026-06~07 起 / 给 5/14 GO-J-ε 后的工作留 1-1.5 个月"沉淀期"。这与 CLAUDE.md §6.2 "抽象优先于完成度"一致 — v0.2 已经达成抽象产出 / v0.3 是学界审稿前精炼 / 不需要急。

**例外**：本月（GO-K-ε）启动 08 v0.3 §8 学术理论锚点首批展开 — 这是 v0.3 的核心扩展点 / 单 session 可完成 / 让 v0.3 启动节奏可控。

## 3. 元反思 — 写作能力短板的实证

按 CLAUDE.md §6.2 "承认能力短板"原则：用户写作偏弱 / 架构师须主动 propose 文档框架 / 起草初稿 / 用户角色更接近"审稿 + 投放"而非"从零写"。

**本月实证（5/8 → 5/14）**：

- 09 v0.1 outline + v0.2 全本（950 行 / 67.2 KB）**完全由架构师（Claude）起草**
- 08 v0.2 章节展开类似
- 用户在以下环节起核心作用：
  - **ground truth 提供**：5 题访谈（Q-006 / Q-020 / Q-026 / Q-027 / Q-028）— F-001 v0.4 命题精细化的必要条件
  - **决策授权**：每个 GO-X 阶段的推进选择 / 双文章配对节奏选择 / P3 X2 决策
  - **战略判断**：D-route §6.2 案例优先级判定 / 写作节奏选择
- **新的协作模式**：架构师做"起草工" + 用户做"审稿 + 决策 + ground truth 提供"

**对未来的启示**：

1. 这个模式在 2026-10 双文章 v1.0 发布前应当**持续保持**
2. 用户审稿环节应当在 v0.3 完整初稿 + v0.4 审稿修订时**显著加重**（v0.3 是学界审稿前的最后形态）
3. 月度决策日记**也由架构师起草 / 用户审稿** — 但每篇日记前应让用户标注"哪些反思我同意 / 哪些需要修正"
