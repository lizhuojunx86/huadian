## 0. 目的与定位

三篇 6/2 当日爆发到 v0.3 齐平（GO-V/W/X-η）。日记 §2.5 已记一条诚实成本：**审稿负荷被前置且叠加**——三篇完整 v0.3 同时摆上审稿台。本计划的作用：

1. 把"审什么"具体化为**每篇的真实弱点靶点**（不是泛泛"找专家看看"）；
2. 把"什么次序"定为**错峰序列**，让作者（用户 + 架构师）带宽不在同一窗口被三篇同时占满；
3. 明确每篇审稿的**交付物**（审稿意见 → v0.4 修订点 → v0.4 lock）。

时间锚：三篇 v0.4 审稿 ETA 2026-09，v1.0 发布锚点 2026-10（D-route §6）。当前领先 ~6-8 周，有错峰余量。

## 3. 错峰次序 + rationale

**提议序列：09 → 08 → 10**（依次进入审稿窗，不并发）。

```
2026-08 上旬   09 审稿窗（结构 / 地基）
2026-08 下旬   08 审稿窗（机制 / 建于 09 之上）
2026-09 上旬   10 审稿窗（检测 / 建于 08 之上 + 最新需沉淀）
2026-09 下旬   三篇 v0.4 lock 收口
2026-10        v1.0 三发布
```

**为什么是 09 → 08 → 10**：

1. **依赖链**：F-001（09）是 F-002/F-003/F-004 的**共同结构性基础**（09 §9.3 明述）——所有方向的反算都栖息在"有不可见中深层"的分层结构里。先把地基（09 的三层定义 + 代差）审定 lock，下游 08（机制栖息于结构）、10（检测 F-002 通道）的审稿才建在锁定的前提上，避免返工。
2. **理论权重**：09 是"对学界的主要理论贡献"（STS / 组织社会学），学理风险最高、最该先啃硬骨头；08 次之（行为经济/会计有成熟对照文献）；10 最技术（数学/算法可自证），可放最后做紧审。
3. **沉淀**：10 是 6/2 当天最新写的、跨校验最少；放最后给它最长的"审稿前沉淀窗"（呼应日记 §2.1 沉淀期工具）。
4. **带宽**：三篇专属审稿人池不同（09 偏 STS/社会学 / 08 偏经济/会计 / 10 偏统计/检测），错峰让用户 + 架构师的**作者侧带宽**不被三篇同时占满（§2.5 核心成本）。

**备选**：若某篇的外部审稿人档期先到，可局部调序——但 09 不宜放到最后（地基后审会让 08/10 的前提悬空）。

## 4. 每篇审稿的交付物 + 流程

每篇走统一三步（仿日记"草稿允许预测，定版负责对账"的闭环）：

1. **审稿意见收集**：按 §1 共通 + §2 专属视角，逐靶点记"同意 / 质疑 / 要补"。用户负责对接真实外部审稿人（行业 / 学界）；架构师负责把意见整理成结构化修订单。
2. **v0.4 修订点清单**：每条审稿意见 → 一个可执行修订（改论点 / 补文献 / 加边界条件 / 弱化过强主张）。citation 卷页最终化是每篇必做项。
3. **v0.4 lock**：修订落地 + 存 v0.3→v0.4 快照（仿 archive 惯例）+ 在 case-2 §10 记 GO 阶段 + 三篇配对 lock。
