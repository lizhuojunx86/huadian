## 详细结果

### 1. 项目宪法合规 (C-1 ~ C-21)

#### 一、数据第一性

- **[PASS] C-1: 一次结构化 N 次衍生**
  无为单个产品建立的独立数据表。33 张业务表构成统一知识层，Web/API/Pipeline 均为查询视图。

- **[PASS] C-2: 可溯源**
  Traceable 接口在 5 个核心实体（Book / SourceEvidence / Person / Event / Place）上实现 sourceEvidenceId + provenanceTier + updatedAt。ADR-009 允许 sourceEvidenceId nullable（Phase 0 数据尚未全面关联 SourceEvidence）。

- **[PASS] C-3: 多源共存**
  EventAccount / AccountConflict 双层建模正确实现（ADR-002）。Event 为抽象锚，EventAccount 为具体叙述。

- **[PASS] C-4: 历史原貌**
  - entity_revisions 审计表存在（migration 0000，含 diff JSONB / actor_type / reason / promptVersion）
  - soft-delete（deletedAt 列）覆盖：persons / events / places / polities / reignEras / relationships / mentions / artifacts / books / sourceEvidences 等主要实体
  - Python pipeline 中无 `DELETE FROM` / `TRUNCATE` / `DROP TABLE` 语句
  - 所有实际删除操作均通过 soft-delete + person_merge_log 审计

- **[PASS] C-5: 单 PostgreSQL**
  未引入任何独立图数据库、向量库或搜索引擎。pgvector + PostGIS + pg_trgm 均在 PG 内部。neo4j / elasticsearch / pinecone 等关键词仅出现在 ADR-001 的"切出触发器"说明中。

- **[PASS] C-6: Schema-first**
  Drizzle schema 是 DDL 唯一真源。所有 API 变更先改 SDL 再实现（CHANGELOG 记录验证）。codegen 确保 SDL → TS types 同步。

- **[PASS] C-7: 无黑盒 LLM**
  所有 LLM 调用通过 `ai/gateway.py` LLMGateway Protocol。AnthropicGateway 记录 prompt_version / prompt_hash / input_hash / model / response / cost / latency。未发现绕过 Gateway 的直接 anthropic SDK 调用。

- **[PASS] C-8: 质检嵌入**
  TraceGuard adapter 完整实现：adapter.py + rule_registry.py（5 条首批规则）+ policy.py + traceguard_policy.yml（三段策略）+ audit.py（双写）+ replay.py。checkpoint 在 extract 管线步骤中被调用。

- **[WARN] C-9: 可观测性先于优化**
  - GraphQLContext 有 `tracer: Tracer | null` 字段（stub）
  - OTel Collector 在 docker-compose 中配置（logging exporter）
  - **但 Python pipeline 和 API 均未接入 OpenTelemetry SDK**
  - Phase 0 可接受，T-P0-005a（SigNoz 接入）已规划
  - **建议**：Phase 1 开始前必须接入 OTel trace

- **[PASS] C-10: 类型端到端**
  Drizzle schema → GraphQL SDL → graphql-codegen TS types → 前端 typed documents。shared-types 提供 zod → JSON Schema → Pydantic 全链路。字段命名一致（Drizzle camelCase TS ↔ GraphQL camelCase ↔ SQL snake_case）。

- **[WARN] C-11: 可回滚**
  - **LLM 抽取**：按 prompt_version 保留历史（extractions_history 表 + idempotent index on paragraph_id/step/prompt_version）✅
  - **迁移 DOWN 方案**：Drizzle 和 pipeline SQL 迁移均为**仅前进**（forward-only），无 DOWN/ROLLBACK 语句 ⚠️
  - Phase 0 表结构简单，可通过手工 SQL 回滚，但随着 schema 复杂化需要正式 down 方案
  - **建议**：Phase 1 前建立迁移回滚规范（或引入 ADR-016 软删除与修订审计策略）

- **[PASS] C-12: 多语言**
  所有 user-facing 文本字段使用 JSONB 多语言结构（MultiLangText），persons.name / places.name / books.title 等均为 `jsonb("name")` 类型。GraphQL 暴露为 MultiLangText object type。

- **[PASS] C-13: URL 稳定**
  所有实体有 slug 字段。API 优先使用 slug（person(slug) / event(slug) / place(slug)）。ADR-011 规范化 slug 生成规则（Tier-S pinyin + unicode fallback），ADR-013 partial unique index 保护。

- **[PASS] C-14: 许可证前置**
  Book 类型有 `license: String!` 字段。ADR-008 规范化 license 枚举（CC_BY / public_domain 等）。
