**Strategy C was selected**: simplest implementation, highest F1, zero violations.

## Key Changes

1. **`similarityThreshold(term)`**: Returns threshold based on query character count:
   - 1-2 chars: 0.5 (strict, avoids short-query trigram FP)
   - 3 chars: 0.4 (moderate)
   - 4+ chars: 0.3 (original, preserves fuzzy recall)

2. **`aliasSubstringSearch()`**: Fallback when `trigramSearch` returns zero results.
   Uses `pn.name ILIKE '%term%'` on person_names only. Prevents FP because it only
   triggers when the primary search found nothing.
